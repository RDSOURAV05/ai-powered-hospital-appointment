MediGraph RAG Architectural Flow

Understand how queries are routed and answered in a state-graph multi-agent architecture.

System Architecture Schematic

1. State Graph Loops

Unlike standard linear RAG pipelines, LangGraph permits cyclic execution where agents can revisit planner stages if context is lacking. The graph holds the medical state in memory across steps, ensuring safe transitions.

2. Dynamic Query Routing

The system uses a LLM-based router (Llama-3.3) to classify queries. Rather than querying the vector database for greetings or unrelated chat, the router sends them directly to the answer node, saving tokens and response time.

3. FAISS Vector Database

FAISS (Facebook AI Similarity Search) is loaded as a vector index. Chunks from the medical clinical manuals are embedded into 384-dimensional dense vectors. During search, cosine similarity maps user intent to database rules.

4. Strict Triage Fallbacks

If similarity search finds no document matching a critical medical query above a strict threshold, the answerer is prohibited from hallucinating clinic fees or fasting guides, instead rendering a safe clinic helpline fallback message.
